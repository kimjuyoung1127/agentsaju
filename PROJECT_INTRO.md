# 사주 라이프 시뮬레이터 (Saju Life Simulator)

> "운명을 읽는 것이 아니라, 시뮬레이션한다."

---

## 한 줄 요약

**사주 명리학의 계산 체계를 결정론적 시뮬레이션 엔진으로 구현하고, 멀티 에이전트 토론을 통해 근거 기반의 인생 예측 리포트를 생성하는 플랫폼.**

---

## 왜 이걸 만드는가?

기존 사주 서비스의 문제는 명확합니다:

| 기존 서비스 | 이 프로젝트 |
|------------|-----------|
| LLM이 사주풀이 "생성" → 매번 다른 결과 | **결정론적 엔진**이 계산 → 같은 입력 = 같은 결과 |
| "올해 재물운이 좋습니다" 식의 모호한 텍스트 | **4개 도메인 × 수치화된 상태** + 구체적 이벤트 |
| 근거 없는 해석 | 모든 예측에 **evidence_ref** 필수 |
| 정적인 1회성 감정 | **분기(Branch) 시뮬레이션**으로 "만약 ~한다면?" 비교 |
| 검증 불가능 | **백테스트** 기능으로 과거 예측 정확도 측정 |

핵심 철학: **LLM은 설명만 한다. 사실(fact)은 룰 엔진이 만든다.**

---

## 어떻게 작동하는가?

```
사용자 생년월일 + 목표 + 제약조건
        ↓
┌─────────────────────────────────────────────┐
│  Layer 1. Chart Engine (결정론적)              │
│  → 사주 원국(ChartCore) + 특성 벡터(FeatureVector) │
│  → 대운/세운/월운/주운 타임라인 팩터              │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│  Layer 2. State Model                        │
│  → 4대 도메인: 커리어 / 관계 / 재물 / 건강      │
│  → 각 0~100 수치 + 스트레스/모멘텀/지지도        │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│  Layer 3. Rule Engine + Simulator            │
│  → 이벤트 후보 생성 (CAREER_MOVE, WEALTH_LEAK 등) │
│  → 베이스라인 + 브랜치 시뮬레이션               │
│  → 모든 이벤트에 evidence_ref 필수              │
└─────────────────┬───────────────────────────┘
                  ↓
┌─────────────────────────────────────────────┐
│  Layer 4. Agent Orchestration + Narrative     │
│  → 7명의 오라클 에이전트가 근거 기반 토론         │
│  → Judge가 월드라인 랭킹                       │
│  → 내러티브 레이어가 리포트 생성                 │
└─────────────────────────────────────────────┘
```

## 7명의 오라클 에이전트

단순히 "사주를 봐주는 AI"가 아닙니다. **각자 다른 관점에서 근거를 들어 토론하는 전문가 패널**입니다.

| 에이전트 | 역할 | 관점 |
|---------|------|------|
| **DestinyAnalyst** | 원국 해석 | 사주 구조 자체의 의미 |
| **CareerOracle** | 직업/경력 | 커리어 도메인 집중 분석 |
| **RelationshipOracle** | 대인관계 | 관계 도메인 집중 분석 |
| **HealthOracle** | 건강/에너지 | 건강 도메인 집중 분석 |
| **RiskOracle** | 리스크 경고 | 하방 리스크 식별 |
| **FreeWillOracle** | 자유의지 옵션 | "이렇게 하면 바뀔 수 있다" |
| **JudgeAgent** | 최종 판정 | 증거 밀도 × 목표 정렬로 랭킹 |

**규칙:** 모든 발언은 `evidence_ref`를 인용해야 합니다. 근거 없는 주장은 무시됩니다.

## 기술 스택

```
Backend:   FastAPI + PostgreSQL (JSONB) + Redis Streams + Celery
Engine:    순수 Python 결정론적 계산 (LLM 미사용)
Agents:    LLM 기반 (Gemini 2.5 Flash) — 설명/토론 전용
Storage:   Append-only event log + lineage tracking
Replay:    seed + policy_snapshot → 완전 재현 가능
```

---

## 차별점 요약

```
1. 결정론적   — 같은 사주 + 같은 시드 = 같은 결과. 항상.
2. 근거 기반   — 모든 예측에 증거 참조 필수. 허풍 불가.
3. 시뮬레이션  — "만약 ~한다면?" 브랜치 비교로 의사결정 지원.
4. 검증 가능   — 백테스트로 과거 예측 정확도 측정.
5. 재현 가능   — seed + lineage로 언제든 동일 결과 재생산.
```
