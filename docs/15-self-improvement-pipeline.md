# 15. 자기개선 파이프라인 & 데이터 전략

Last updated: `2026-03-18`

## 핵심 원칙

"LLM을 막연히 재학습"이 아니라,
**이벤트 스키마 정의 → 종단 데이터 확보 → 룰/상태전이/합의 프로토콜을 백테스트로 보정**하는 것.

자기개선 대상은 LLM이 아니라 **시스템 파라미터**다:
- 룰 가중치
- 상태 전이식
- 도메인별 이벤트 확률
- 에이전트 합의 프로토콜
- 설명 품질

---

## 1. 정답 정의: 이벤트 스키마

사주 해석 문장을 정답으로 쓰지 않는다.
**사건(event)과 상태(state)**로 변환해야 한다.

### 좋은 정답 변수

| 변수 | 유형 | 예시 |
| --- | --- | --- |
| `education_level` | 상태 | 고졸/대졸/석사 |
| `first_job_age` | 이벤트 | 22세 |
| `unemployment_spell` | 이벤트 | 28~29세 실업 기간 |
| `marriage` | 이벤트 | 31세 결혼 |
| `divorce` | 이벤트 | 38세 이혼 |
| `childbirth` | 이벤트 | 33세 첫 자녀 |
| `relocation` | 이벤트 | 25세 지역 이동 |
| `major_health_event` | 이벤트 | 45세 입원 |
| `business_start` | 이벤트 | 35세 창업 |
| `income_decile_change` | 상태 | 5분위 → 7분위 |

### 나쁜 정답 (사용 금지)

- "성공했다", "인간관계가 좋았다", "재물운이 강했다"
- 추상적이라 자동 평가 불가

## 2. 데이터 소스 (우선순위순)

### 2.1 공개 종단 패널 데이터 (1순위)

가장 현실적. 표본 크고, 실제 추적 데이터, 결과변수 있음.

**미국:**

| 데이터셋 | 시작년 | 특징 |
| --- | --- | --- |
| **PSID** | 1968 | 세계 최장수 가구 패널. 고용/소득/부/건강/교육/결혼/출산 |
| **NLSY79 / NLSY97** | 1979/1997 | 청년 코호트 장기 추적. NLSY97: 8,984명, 12~16세부터, 20라운드 |
| **Add Health** | 1994 | 7~12학년부터 성인기까지 추적. 사회/경제/심리/건강. 6웨이브 |

**한국:**

| 데이터셋 | 특징 |
| --- | --- |
| **청년패널조사 (KEIS YP)** | 학교→직업 이행, 노동시장 경로 |
| **한국청소년패널조사 (KYPS)** | 동일 표본 반복 조사 종단 설계 |
| **KCYPS** | 아동/청소년 성장 발달 종합 패널 |
| **KLoSA** | 고령화연구패널 (생애 후반부, 10세 시작 예측에는 부적합) |

**한계:** 정확한 출생시각이 없음 → 시주 계산 불가 → coarse model용

### 2.2 공개 인물 전기/위키 (2순위 — 데모용)

| 소스 | 특징 |
| --- | --- |
| **Wikidata** | 자유 오픈 지식베이스. 출생일/출생지/직업/가족관계 |
| **DBpedia** | 위키백과 인포박스 구조화. birthDate/birthPlace/spouse/occupation |

**용도:** 초기 프로토타입 벤치마크, 스토리텔링/쇼케이스
**한계:** 유명인 편향, 출생시각 없음, 일반화 불가

### 2.3 사용자 동의 기반 자체 데이터 (3순위 — 장기 핵심)

수집 항목:
- 생년월일 / 출생시각 / 출생지
- 과거 주요 사건 체크리스트
- 학력, 첫 취업, 결혼, 이직, 질병, 이사, 자녀, 창업 타임라인
- 익명화 저장
- 백테스트 허용 동의

**장점:** 사주 계산 해상도 확보, 한국식 해석에 맞는 로컬 데이터
**단점:** 초기 데이터 적음, 동의/보안/IRB 설계 필요, 자기보고 편향

### 2.4 전문가 라벨링 + 합성 데이터 (보조)

- 전문가가 차트별 위험 신호 라벨링
- 규칙 엔진으로 synthetic chart 생성
- 특정 사건 분포를 가진 가짜 생애 경로 생성
- 에이전트 토론 결과를 품질검사용 보조정답으로 사용

**주의:** 최종 검증은 반드시 실제 데이터로.

## 3. 평가 파이프라인

### 3.1 관측 시점 고정

미래 정보 누수(leakage)를 방지한다.

| 관측 시점 | 사용 가능 정보 | 예측 대상 |
| --- | --- | --- |
| 10세 | 출생정보 + 가정환경 | 11~40세 사건 |
| 15세 | + 초기 학업 | 16~40세 사건 |
| 20세 | + 교육/첫 직업 | 21~40세 사건 |

### 3.2 예측 출력 표준화

모델 출력 형식:

```json
{
  "predictions": [
    { "event": "first_job", "before_age": 25, "probability": 0.74 },
    { "event": "marriage", "before_age": 30, "probability": 0.42 },
    { "event": "relocation", "age_range": [20, 30], "probability": 0.61 }
  ]
}
```

### 3.3 평가 지표

| 지표 | 용도 |
| --- | --- |
| Brier score | 확률 정확도 |
| Calibration curve | 확률 캘리브레이션 |
| Top-k recall | 상위 예측 적중률 |
| 기간 오차 (MAE) | 시점 정확도 |
| Domain-wise AUROC/F1 | 도메인별 성능 분리 |

### 3.4 실패 원인 분해

성능이 나쁠 때 **어디가 틀렸는지** 분리:

| 계층 | 점검 항목 |
| --- | --- |
| 차트 계산 | 만세력/절기/시주 계산 오류 |
| 룰 가중치 | 특정 규칙의 과대/과소 반영 |
| 상태 전이식 | 도메인 간 에너지 전이 오차 |
| 에이전트 합의 | 합의 프로토콜 불안정 |
| 설명 품질 | 예측은 맞는데 설명이 틀림 (또는 그 반대) |

## 4. 자동화 범위

### 자동화 가능 (강하게)

- 백테스트 실행
- 오류 집계
- 도메인별 calibration 계산
- 근거 부족 주장 탐지
- 규칙 후보 제안
- 에이전트 프롬프트 후보 비교
- branch usefulness 평가

### 자동화 주의 (보수적으로)

- 핵심 사주 룰 자체를 완전 자동 수정
- 전문가 검토 없이 용신/희신 판정 로직 교체
- 소수 사례로 룰 일반화

> **원칙:** 자동화는 "제안과 평가"는 강하게, "핵심 규칙 교체"는 보수적으로.

## 5. 출생시각 부재 대응

| 방법 | 설명 | 용도 |
| --- | --- | --- |
| 연/월/일만 사용 | 시주 의존 특성은 낮은 가중치 | 공개 데이터용 |
| 시주 latent variable | 12개 후보 모두 돌리고 불확실성 반영 | 공개 데이터 정밀화 |
| 정밀 사주 모델 | 출생시각 포함 자체 데이터만 사용 | 자체 데이터용 |

**이중 구조:** 공개 데이터 → coarse model, 자체 데이터 → full model

## 6. 데이터 확보 로드맵

| 단계 | 작업 | 데이터 소스 |
| --- | --- | --- |
| 1단계 | 이벤트 예측기/평가기 프레임 구축 | PSID, NLSY, Add Health, 한국 패널 |
| 2단계 | 전기형 데모 데이터셋 구축 | Wikidata, DBpedia |
| 3단계 | 한국형 사주 데이터셋 구축 | 자체 동의 기반 (출생시각 포함) |
| 4단계 | 이중 검증 | 공개 → 일반 예측 안정성, 자체 → 사주 특화 정밀도 |

## 7. 핵심 경고

### "한 사람 깊이" < "많은 사람 같은 질문"

- 한 사람의 전 생애를 자세히 돌리는 건 케이스 스터디용
- 모델 개선은: **많은 사람 × 같은 이벤트 정의 × 같은 평가 기준 × 반복 측정**
- "아인슈타인 인생을 얼마나 잘 맞췄나" < "10,000명의 30세 이전 결혼/이직/질병 캘리브레이션"

### 유명인 전기의 위치

- 데모에는 좋음
- 학습/평가의 중심은 패널 데이터 또는 자체 동의 데이터
