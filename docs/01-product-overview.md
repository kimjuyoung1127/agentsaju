# 01. Product Overview

## Product Definition

이 서비스는 사용자의 출생 정보와 현재 목표를 입력받아, 사주 기반 상태 변수와 사건 규칙을 통해 월/주 단위 인생 세계선을 시뮬레이션하는 플랫폼이다.

최종 출력은 다음 세 가지를 동시에 만족해야 한다.

1. 무엇이 일어날 가능성이 높은지 보여준다.
2. 왜 그런 결론이 나왔는지 근거를 보여준다.
3. 어떤 선택이 더 나은 분기를 만드는지 비교해 준다.

## Target Users

### Primary

- 이직, 퇴사, 연애, 재회, 동업, 투자 같은 갈림길 앞에 있는 20~40대 사용자
- 텍스트 운세보다 “내 상황에 맞는 분기 비교”를 원하는 사용자

### Secondary

- 궁합, 관계성, 팀워크를 보고 싶은 2인 이상 사용자
- 과거 연도 백테스트로 엔진 신뢰도를 확인하고 싶은 사용자

## Core User Journeys

### Journey A: 개인 시뮬레이션

1. 출생 정보와 현재 목표를 입력한다.
2. 차트와 feature vector를 생성한다.
3. 12개월 baseline simulation을 실행한다.
4. 월별 주요 사건과 4대 자원 변화를 본다.
5. 특정 월을 클릭해 주 단위 확대 또는 선택지 비교를 수행한다.

### Journey B: 결정 실험실

1. 사용자가 선택지를 입력한다.
2. 시스템이 동일 차트로 branch runs를 생성한다.
3. 각 분기에서 달라지는 사건, 득실, 리스크를 비교한다.

### Journey C: 설명 확인

1. 사용자가 특정 사건 카드를 클릭한다.
2. 시스템이 근거 규칙, 상태 변화, 관련 이벤트를 보여준다.
3. Narrative layer가 근거를 바탕으로 설명과 행동 스크립트를 만든다.

## Product Boundaries

### Included

- 개인 사주 차트 계산
- 4대 자원 기반 상태 시뮬레이션
- 월 기본, 주 확대 시뮬레이션
- 근거 기반 리포트
- 멀티에이전트 토론 기반 세계선 집계

### Excluded from Core

- 결제
- 대시보드 UI
- 전문가 상담 연결
- 마케팅 자동화

위 항목은 모두 상위 오케스트레이션 레이어이며, 코어 문서에서는 코어 소비 방식만 정의한다.

## Free vs Paid Product Framing

### Free

- 기본 차트 요약
- 연간 히트맵
- 상위 사건 3개 요약

### Paid

- 특정 월 주 단위 확대
- branch comparison
- evidence console
- action scripts
- backtest report

## Success Criteria

- 같은 입력과 같은 정책에서 같은 차트가 생성된다.
- 같은 `seed`에서 같은 시뮬레이션 결과가 재현된다.
- 각 사건에는 `evidence_refs`가 존재한다.
- LLM 응답은 근거 없는 새 사실을 만들지 않는다.
- 사용자는 “왜 이런 결과가 나왔는지”를 추적할 수 있다.
