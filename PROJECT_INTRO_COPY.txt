사주 라이프 시뮬레이터 — "운명을 읽는 것이 아니라, 시뮬레이션한다."

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

사주 명리학의 계산 체계를 결정론적 시뮬레이션 엔진으로 구현하고,
멀티 에이전트 토론을 통해 근거 기반의 인생 예측 리포트를 생성하는 플랫폼입니다.


[왜 만드는가?]

기존 사주 서비스는 LLM이 "운세 텍스트"를 생성합니다.
매번 다른 결과가 나오고, 근거도 없고, 검증도 불가능합니다.

이 프로젝트는 다릅니다:

• 결정론적 — 같은 사주 + 같은 시드 = 같은 결과. 항상.
• 근거 기반 — 모든 예측에 증거 참조(evidence_ref) 필수. 허풍 불가.
• 시뮬레이션 — "이직하면?" vs "잔류하면?" 브랜치 비교로 의사결정 지원.
• 검증 가능 — 백테스트로 과거 예측 정확도 측정.
• 재현 가능 — seed + lineage로 언제든 동일 결과 재생산.

핵심 철학: LLM은 설명만 한다. 사실(fact)은 룰 엔진이 만든다.


[어떻게 작동하는가?]

4단계 파이프라인으로 돌아갑니다.

1단계 — Chart Engine (결정론적 계산)
  생년월일을 입력하면 사주 원국(ChartCore)과 특성 벡터(FeatureVector)를 계산합니다.
  대운, 세운, 월운, 주운 타임라인 팩터도 여기서 나옵니다.
  LLM 개입 없음. 순수 룰 기반 계산.

2단계 — State Model (인생 상태 수치화)
  커리어 / 관계 / 재물 / 건강 — 4개 도메인을 각각 0~100으로 추적합니다.
  스트레스, 모멘텀, 지지도 같은 보조 지표도 함께 관리합니다.

3단계 — Rule Engine + Simulator (이벤트 생성 & 시뮬레이션)
  타임라인 팩터 + 현재 상태를 기반으로 이벤트 후보를 생성합니다.
  (예: CAREER_MOVE, HEALTH_ALERT, WEALTH_GAIN 등)
  베이스라인 시뮬레이션 + "만약 ~한다면?" 브랜치 시뮬레이션을 돌립니다.
  모든 이벤트에는 evidence_ref가 필수입니다.

4단계 — Agent Orchestration + Narrative (토론 & 리포트)
  7명의 오라클 에이전트가 근거를 들어 토론합니다.
  Judge가 최종 월드라인을 랭킹하고, 내러티브 레이어가 리포트를 생성합니다.


[7명의 오라클 에이전트]

단순히 "사주를 봐주는 AI"가 아닙니다.
각자 다른 관점에서 근거를 들어 토론하는 전문가 패널입니다.

• DestinyAnalyst — 사주 원국 구조 자체의 의미 해석
• CareerOracle — 커리어 도메인 집중 분석
• RelationshipOracle — 관계 도메인 집중 분석
• HealthOracle — 건강/에너지 도메인 집중 분석
• RiskOracle — 하방 리스크 식별 및 경고
• FreeWillOracle — "이렇게 하면 바뀔 수 있다" 자유의지 옵션 제시
• JudgeAgent — 증거 밀도 × 목표 정렬로 최종 랭킹 판정

규칙: 모든 발언은 evidence_ref를 인용해야 합니다. 근거 없는 주장은 무시됩니다.


[실제 결과물 예시]

2026년 3월 시뮬레이션 결과

  도메인 상태:
    커리어  78/100  (+12)
    관계    62/100  (-3)
    재물    71/100  (+5)
    건강    54/100  (-8)

  핵심 이벤트:
    1. CAREER_MOVE (중요도 87, 신뢰도 0.82)
       "편관+식신 조합이 세운 을사와 충돌 → 직무 변경 압력"
       근거: rule_035, feature_clash_penalty

    2. HEALTH_ALERT (중요도 72, 신뢰도 0.71)
       "화 과다 + 월운 병오 → 과로 경고"
       근거: rule_088, element_imbalance

    3. WEALTH_GAIN (중요도 65, 신뢰도 0.68)
       "정재 투출 + 대운 지지 → 부수입 기회"
       근거: rule_052, timeline_support

  브랜치 비교 — "이직 vs 잔류":
    이직 시: 커리어 +18, 관계 -12, 재물 -5
    잔류 시: 커리어 +3, 관계 +5, 재물 +8
    → 목표가 "커리어 성장"이면 이직, "안정"이면 잔류 우세

  액션 스크립트:
    • 3월 2~3주차: 네트워킹 강화 (CareerOracle 권고)
    • 3월 전체: 수면 관리 필수 (HealthOracle 경고)
    • 재테크 의사결정은 4월로 지연 권고 (RiskOracle)


[기술 스택]

• Backend: FastAPI + PostgreSQL(JSONB) + Redis Streams + Celery
• Engine: 순수 Python 결정론적 계산 (LLM 미사용)
• Agents: LLM 기반 (Gemini 2.5 Flash) — 설명/토론 전용
• Storage: Append-only event log + lineage tracking
• Replay: seed + policy_snapshot → 완전 재현 가능


[타겟 사용자]

• 20~40대, 중요한 결정(이직, 연애, 투자) 앞에 선 사람
• "운세"가 아니라 "시뮬레이션 기반 의사결정 도구"를 원하는 사람
• 커플 궁합 비교, 백테스트 검증을 원하는 심화 사용자


[수익 모델]

무료: 기본 사주 원국 / 연간 히트맵 / TOP 3 이벤트 요약
유료: 월간 상세 줌 / 브랜치 비교 시뮬레이션 / 증거 콘솔 / 액션 스크립트 / 백테스트
