# 사주 라이프 시뮬레이터 Wireframe / UX Spec

Last updated: `2026-03-17 (KST)`  
Product lens: `운세 텍스트 앱`이 아니라 `미래를 관찰하는 시뮬레이터`

## 1. 경험 목표
- 사용자는 접속 후 5초 안에 "이 서비스는 내 1년을 시뮬레이션해 보여준다"는 점을 이해해야 한다.
- 복잡한 명리 해석은 뒤로 숨기고, 먼저 `연간 흐름`, `변곡점`, `행동 지침`이 보이게 해야 한다.
- 무료 상태에서도 "어떤 일이 생길지"는 어느 정도 이해되지만, 결제 후에만 `왜`, `얼마나`, `어떻게 피할지`가 열린다.
- 디자인은 싸구려 신비주의가 아니라 `개인화된 미래 관제실`처럼 보여야 한다.

## 2. 비주얼 방향

### 2.1 룩앤필
- Tone: `Warm Observatory + Personal ERP`
- 인상: 정교함, 신뢰감, 약간의 운명성, 과장 없는 긴장감
- 피해야 할 것: 보라색 위주 신비주의, 별/은하 남발, 과한 점술 클립아트

### 2.2 컬러
- Background: `#F4EEE6`
- Surface: `#FFF9F1`
- Primary Ink: `#1F1A17`
- Line/Borders: `#D6C6B4`
- Primary Accent: `#8B5E34`
- Secondary Accent: `#2F6F66`
- Danger: `#B5523B`
- Warning: `#D89A3D`
- Positive: `#4E7A49`

의미 연결:
- `갈색/황동 계열`은 운명/기록/연대기
- `청록`은 기회/완충
- `적갈색`은 충돌/경고

### 2.3 타이포그래피
- Heading: `MaruBuri` 또는 동급의 명조 계열
- Body/UI: `SUIT Variable`
- Numeric/Timeline: `IBM Plex Sans`

## 4. 글로벌 UI 규칙

### 4.1 상단 네비게이션
- 좌측: 로고, 현재 프로필 이름
- 중앙: `연간 시뮬레이션`, `리포트`, `백테스트`, `설정`
- 우측: 플랜 배지, 알림, 프로필 메뉴

### 4.2 도메인 게이지
모든 앱 화면 상단에 4개 리소스 게이지를 반복 사용한다.

- `재물`
- `커리어`
- `관계`
- `건강`

각 게이지는 다음을 포함한다.
- 현재 값
- 전월 대비 변화
- 위험/기회 컬러
- 클릭 시 상세 원인 툴팁

### 4.3 사건 카드
- 사건 제목
- 한 줄 티저
- 도메인 배지
- 긴장도/기회도 미니 바
- 무료/유료 상태
- `근거 보기` 또는 `상세 해금` CTA

### 4.4 근거 카드
- rule_id 또는 해석 라벨
- 관련 오행/십성/합충 배지
- 왜 이 사건이 선택되었는지 한 줄 설명
- 관련 로그/이벤트 링크

### 4.5 진행 스트림
컴파일, 월 확대, 리포트 생성 중에는 우측 패널이나 모달에서 실시간으로 보여준다.

- 현재 단계
- 최근 생성 이벤트
- 예상 남은 시간
- 오류 시 재시도 버튼

## 5. 핵심 화면

### 5.1 Landing
목적: "텍스트 운세가 아니라 시뮬레이터"라는 인식을 즉시 만든다.

핵심 블록:
- Hero 카피
- 시뮬레이션 예시 타임라인
- 4도메인 게이지 데모
- "9월 클라이맥스" 카드 예시
- 무료/유료 비교
- 신뢰 요소: 설명 가능성, 정책 공개, 개인정보 보호

권장 카피:
- 메인: `내 AI 아바타가 올해를 먼저 살아보고, 변곡점을 알려줍니다`
- 서브: `사주를 정적인 해설이 아니라 월별 사건과 선택지로 보여주는 라이프 시뮬레이터`

데스크톱 구조:

```txt
+---------------------------------------------------------------+
| Logo | Pricing | Sample Report | FAQ | Login                  |
+---------------------------------------------------------------+
| Headline                               | Live Timeline Preview |
| Subcopy                                | 12-month strip        |
| [무료로 내 흐름 보기] [샘플 리포트]     | Climax card           |
+---------------------------------------------------------------+
| Why this is different (3 cards)                             |
+---------------------------------------------------------------+
| Free vs Paid comparison                                     |
+---------------------------------------------------------------+
| FAQ / Safety / Privacy                                      |
+---------------------------------------------------------------+
```

### 5.2 Birth Input Wizard
목적: 복잡한 입력을 실수 없이 받고, 계산 정책을 숨기지 않으면서도 부담을 줄인다.

스텝:
1. 출생 정보
2. 목표/질문
3. 계산 정책/동의

좌우 레이아웃:

```txt
+-----------------------------+--------------------------------+
| Stepper                     | Preview Panel                  |
|                             | - 입력 요약                    |
| [생년월일시 입력 폼]        | - 선택한 질문 카테고리         |
| [출생지 검색]               | - 개인정보 처리 안내           |
| [양력/음력 토글]            | - 계산 정책 도움말             |
| [성별 / 윤달 / 시간대]      |                                |
|                             |                                |
|               [다음 단계]   |                                |
+-----------------------------+--------------------------------+
```

필수 UX 포인트:
- `시간을 모름` 옵션 제공
- `태양시/야자시`는 기본 감춤 + "고급 옵션 보기"
- 양력/음력 선택이 바뀌면 즉시 포맷 보정
- 질문 카테고리는 카드형 선택: `이직`, `연애`, `재물`, `건강`, `자유 질문`
- 선택 입력으로 `현재 직업`, `관계 상태`, `거주 형태`, `자산 대역`, `MBTI`를 받아 동일 명식 사용자 보정에 활용

### 5.3 Compile Progress
목적: 기다림을 불안이 아닌 기대감으로 바꾼다.

표시 요소:
- 단계 진행 바: `입력 검증 -> 차트 컴파일 -> 월 타임라인 생성 -> 사건 정렬 -> 리포트 준비`
- 실시간 로그 요약
- 생성된 첫 사건 미리보기

와이어:

```txt
+---------------------------------------------------------------+
| "2026년 흐름을 계산하는 중"                                   |
| [##########------] 68%                                        |
| Step list                                                     |
| - 차트 컴파일 완료                                             |
| - 12개월 계수 생성 중                                          |
| - 상위 사건 추리는 중                                          |
|                                                               |
| Recent stream                                                 |
| - 3월 관계 긴장도 상승 감지                                    |
| - 8월 커리어 기회 후보 생성                                    |
+---------------------------------------------------------------+
```

### 5.4 Annual Dashboard
제품의 심장. 사용자는 여기서 "내 1년"을 본다.

권장 구조:

```txt
+---------------------------------------------------------------+
| Profile Summary | Year Switcher | Plan Badge | Run Status     |
+---------------------------------------------------------------+
| Wealth | Career | Relationship | Health Gauges              |
+-------------------------------+-------------------------------+
| Annual Timeline (8 cols)      | Right Rail (4 cols)          |
| - 12 month cards              | - Climax card                |
| - tension/opportunity line    | - This month summary         |
| - monthly domain heat         | - Locked detail CTA          |
| - click for zoom              | - Recent evidence            |
+-------------------------------+-------------------------------+
| Bottom Area                                                    |
| - Event list by month                                          |
| - Evidence cards                                               |
| - Optional diary sync widget                                   |
+---------------------------------------------------------------+
```

반드시 보여야 하는 정보:
- 현재 프로필과 질문 카테고리
- 12개월 타임라인
- 월별 상단 사건 카드 1~3개
- 4도메인 게이지 변화
- 가장 중요한 클라이맥스 이벤트

상호작용:
- 월 카드 hover/click -> 우측 레일 업데이트
- `주 단위 확대` CTA -> 결제 또는 확대 실행
- 근거 카드 클릭 -> 모달/드로어 열기

### 5.5 Month Zoom Drawer
유료 핵심 기능. 월 단위를 주 단위로 쪼개 "실제로 무슨 일이 벌어질지"를 체감하게 한다.

구성:
- 월 헤더
- 주별 긴장도/기회도 미니 차트
- 주별 사건 목록
- 회피 루트 / 대화 스크립트 / 좋은 선택 예시

와이어:

```txt
+--------------------------------------------------+
| 2026년 9월 확대 보기                             |
| [W1] [W2] [W3] [W4] [W5]                         |
|--------------------------------------------------|
| Week 2: 계약 스트레스 급증                        |
| - 원인: 금전/관계 충돌                           |
| - 예상 영향: 재물 -18, 건강 -3                  |
| - 교환비: 지출을 막으면 관계 피로가 늘 수 있음   |
| - 회피 루트: 지출 결정 48시간 유예               |
| - 대화 스크립트: 상사/파트너 대응                |
+--------------------------------------------------+
```

### 5.6 Report Screen
목적: 대시보드에서 본 사건들을 긴 호흡의 해설로 연결한다.

구성:
- 좌측 섹션 내비게이션: `연간 개요`, `커리어`, `관계`, `재물`, `건강`
- 본문: 무료/유료 구간 혼합
- 우측: 근거 카드, 관련 사건 링크, 결제 CTA

리포트는 반드시 `섹션 기반`이어야 하며, 각 섹션은 다음 구조를 가진다.
- 티저 요약
- 핵심 사건 묶음
- 근거 카드
- 교환비 요약
- 행동 지침
- 주 확대 연결 링크

### 5.7 Paywall / Checkout
페이월은 화면을 막는 장벽이 아니라 "설명 가능한 가장 흥미로운 지점"에서 등장해야 한다.

트리거 위치:
- 클라이맥스 카드 클릭 시
- 월 확대 실행 시
- 리포트의 상세 대안 영역 진입 시

구성:
- 지금 잠긴 내용이 무엇인지 명확히 설명
- 열리는 항목: `상세 근거`, `회피 루트`, `주 단위 확대`, `대화 스크립트`
- 요금제 비교: 단건 구매 vs 구독
- 신뢰 문구: "무조건적인 예언이 아니라 설명 가능한 시뮬레이션 해설"

### 5.8 Backtest
목적: 정확도 체감을 높이고 재방문 루프를 만든다.

화면 구성:
- 연도 선택
- 월별 사건 후보 카드
- 사용자의 발생 여부 라벨 버튼
- 강도 슬라이더
- 개인 캘리브레이션 요약

와이어:

```txt
+---------------------------------------------------------------+
| 2023 백테스트                                                 |
| [연도 선택]                                                   |
|---------------------------------------------------------------|
| 5월 커리어 변동 가능성                                        |
| [실제 있었다] [없었다] [애매하다] [기억 안 난다]             |
| 강도: 1 --- 5                                                 |
| 메모: ______________________                                  |
+---------------------------------------------------------------+
```

### 5.9 Synastry Screen (Phase 2)
2인 시뮬레이션용.

핵심 요소:
- 두 프로필 비교 카드
- 시너지/충돌 막대
- 공동 타임라인
- 관점 전환 토글: `내 시점`, `상대 시점`, `중립 시점`

### 5.10 Settings / Privacy
민감 서비스인 만큼 별도 화면이 필요하다.

반드시 포함:
- 저장된 출생 정보 확인
- 데이터 삭제 요청
- 다운로드 요청
- 동의 내역
- 결제 관리

### 5.11 Notification Center / Share Sheet
출시 직후 확장 기능이지만 정보 구조상 미리 자리 잡아야 한다.

구성:
- 인앱 알림 목록: `이번 달 준비 완료`, `클라이맥스 경고`, `백테스트 리마인더`, `결제 만료 안내`
- 알림 설정: 푸시/이메일/인앱 토글, 조용한 시간대 설정
- 공유 시트: 연간 요약 이미지, 클라이맥스 카드 이미지, 개인정보 마스킹 옵션

와이어:

```txt
+---------------------------------------------------------------+
| Alerts                                                        |
| [이번 달 9월 변곡점이 열렸습니다]  [상세 보기]                |
| [백테스트를 완료하면 정확도 보정이 개선됩니다] [이동]         |
|---------------------------------------------------------------|
| Share                                                         |
| [연간 요약 이미지 생성] [클라이맥스 카드 저장]                |
| [이름 숨기기] [출생정보 숨기기]                               |
+---------------------------------------------------------------+
```

## 6. 컴포넌트 상세

### 6.1 Timeline Month Card
- 월 이름
- 긴장도와 기회도 이중 막대
- 상위 도메인 칩 1~2개
- 대표 사건 제목
- 잠김/열림 상태

### 6.2 Climax Card
- "올해 가장 주의할 변곡점"
- 월/주 표시
- 도메인 아이콘
- teaser 2줄
- `상세 원인 보기` CTA

### 6.3 Domain Gauge
- 0~100 범위
- 이전 버킷 대비 화살표
- 색상 변화
- 클릭 시 원인 팝오버

### 6.4 Evidence Card
- rule id 또는 해석 레이블
- 관련 오행/십성 태그
- 짧은 설명
- 연결된 사건 목록

### 6.5 Daily Sync Mini Card
- 오늘 기분/스트레스 빠른 입력
- 주간 평균 미리보기
- 선택 기능임을 명확히 표시

### 6.6 Tradeoff Strip
- `얻는 것`과 `잃는 것`을 한 줄로 병치
- 재물/건강/관계/커리어 아이콘 사용
- 페이월 직전에도 teaser 형태로 노출 가능

### 6.7 Notification Item
- 알림 유형 배지
- 제목
- 요약 문장
- 연결 CTA

### 6.8 Share Summary Card
- 연도/질문 카테고리
- 상위 도메인 1개
- 대표 변곡점 1개
- 개인정보 마스킹 여부

## 7. 상태 설계

### 7.1 Empty
- 차트가 없는 사용자
- 첫 질문 입력 유도

### 7.2 Running
- 진행률
- 스트리밍 로그
- 이탈 방지 문구

### 7.3 Partial Lock
- 티저는 보이되 상세는 잠김
- "무엇이 잠겨 있는지"를 구체적으로 설명

### 7.4 Error
- 입력 오류
- 계산 실패
- 결제 실패
- 확대 생성 실패

모든 에러는 `다음 행동`을 제시해야 한다.

## 8. 반응형 규칙

### Desktop
- 대시보드 12컬럼
- 우측 레일 상시 노출
- 월 카드 hover 상호작용 사용

### Tablet
- 우측 레일을 하단 섹션으로 이동
- 월 확대는 전체 폭 드로어

### Mobile
- 상단 게이지는 가로 스크롤
- 타임라인은 세로 스택 + 월별 아코디언
- 클라이맥스 카드와 결제 CTA를 상단 고정
- 근거 카드는 바텀시트
- 알림 센터와 공유 시트는 풀스크린 시트로 처리

## 9. 마이크로카피 가이드
- 좋은 문장: `9월에는 재정적 압박이 커질 수 있어 지출 결정을 늦추는 편이 안전합니다.`
- 피해야 할 문장: `9월에 무조건 망합니다.`
- 좋은 CTA: `상세 근거 보기`, `이 달 확대하기`, `회피 루트 열기`
- 피해야 할 CTA: `운명 확인`, `절대 결과 보기`

## 10. 최종 UX 판단 기준
이 와이어프레임은 다음 질문에 모두 "예"라고 답할 수 있어야 한다.

- 처음 보는 사용자도 `연간 타임라인`과 `클라이맥스 카드`만으로 제품 가치를 이해하는가?
